### 3. Database Schema Updates (`database/schema.sql`)
Added the **imports** table for upload metadata tracking:

```sql
CREATE TABLE imports (
    id INT PRIMARY KEY AUTO_INCREMENT,
    connection_id INT NOT NULL,
    file_name VARCHAR(255) NOT NULL,
    file_size BIGINT DEFAULT 0,
    columns_count INT DEFAULT 0,
    rows_count INT DEFAULT 0,
    upload_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    status ENUM('ready', 'mapping_in_progress', 'importing', 'completed', 'failed') NOT NULL DEFAULT 'ready',
    error_message TEXT,
    records_imported INT DEFAULT 0,
    records_failed INT DEFAULT 0,
    duration_ms INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (connection_id) REFERENCES connections(id) ON DELETE CASCADE
);
```

## 🗄️ Database Schema Overview

### Core Tables:

| Table | Purpose | Key Fields |
|-------|---------|------------|
| **connections** | Store remote MSSQL server configs | id, name, host, port, database, username, password |
| **imports** | Upload metadata & import job tracking | id, connection_id, file_name, status, records_imported/failed |
| **import_history** | Detailed import execution logs | connection_id, file_type, total_rows, imported_count, failed_count |
| **column_mappings** | Excel → Database column mappings | import_id, excel_column, db_column, data_type |
| **import_logs** | Row-level import audit trail | import_id, row_number, excel_data, status |

### Table Relationships:
```
connections (1) ──────── (M) imports (N) ──── (M) column_mappings (N)
                        │
                        ▼
                 import_logs (N)
                        │
                        └─── (1) ───────── (M) import_history (via connection_id)
```

## 🔌 API Endpoints Summary

### File Upload & Parsing
```http
POST /api/imports/upload
Content-Type: multipart/form-data

Body: 
  file: [Excel or CSV file]
  connection_id: <number>

Response: {
  "success": true,
  "data": {
    "file_id": "<auto-generated>",
    "file_name": "employees.csv",
    "columns_count": 8,
    "rows_count": 1500,
    "column_names": ["Employee_ID", ...],
    "sample_rows": [...]
  }
}
```

### Column Mapping
```http
POST /api/imports/mapping
Content-Type: application/json

Body: {
  "import_id": "<from upload response>",
  "file_id": "<optional>",
  "mappings": {
    "Employee_ID": {"field": "emp_id", "transform": null},
    "Full_Name": {"field": "full_name", "transform": null}
  }
}

Response: {
  "success": true,
  "data": {
    "mapping_id": <timestamp>,
    "columns_mapped": 2
  }
}
```

### Execute Import
```http
POST /api/imports/execute
Content-Type: application/json

Body: {
  "import_id": "<uuid>",
  "file_id": "<optional>",
  "connection_id": <number>,
  "target_table": "employees"
}

Response: {
  "success": true,
  "data": {
    "records_total": 1500,
    "records_imported": 1500,
    "records_failed": 0,
    "duration_ms": 234
  }
}
```

### Import History & Details
```http
GET /api/imports/history?limit=10&offset=0&status=ready
GET /api/imports/:id
DELETE /api/imports/:id
```
